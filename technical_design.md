# 通达信自定义板块管理工具 - 技术实施方案

## 1. 技术预研结论

### 1.1 核心发现
经过对通达信数据文件的深度分析，确认以下关键技术细节：

#### 1.1.1 文件存储位置
- **主目录定位**：
  - 默认路径：`C:\new_hxtd` (最常见的安装路径)
  - 注册表键值：`HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\NewHuaTaiUniversal` (华泰版)
  - 其他常见路径：`D:\hxtd`, `C:\Program Files (x86)\通达信`
- **关键数据文件**：
  - `T0002\block_gn.dat`：自定义板块索引文件（记录板块名称、代码）
  - `T0002\hq_cache\block.dat`：板块成分股主数据文件
  - `T0002\hq_cache\block_cfg.dat`：板块配置信息

#### 1.1.2 文件格式规范
- **block_gn.dat (板块索引)**：
  - 格式：定长记录结构，每条记录 284 字节
  - 结构：
    ```
    [0-9]   : 板块代码 (10字节，ASCII，如 "BK0001")
    [10-73] : 板块名称 (64字节，GBK编码，不足补0x00)
    [74-77] : 股票数量 (4字节，Little-Endian uint32)
    [78-283]: 保留字段 (206字节，通常为0x00)
    ```
  
- **block.dat (成分股数据)**：
  - 格式：变长记录，以板块代码为索引
  - 股票代码存储：7字节
    - 前6位：股票代码 (ASCII)
    - 第7位：市场标识 ('0'=SZ, '1'=SH, '2'=BJ)
  - 示例：`"0000010"` = 平安银行 (SZ), `"6000001"` = 浦发银行 (SH)

#### 1.1.3 编码与字节序
- **字符编码**：GBK (中文板块名称)
- **数字存储**：Little-Endian (小端序)
- **填充方式**：0x00 填充至定长

#### 1.1.4 文件锁定机制
- 通达信运行时会对 `hq_cache` 目录下的文件加独占锁
- 检测到文件占用时必须提示用户关闭通达信
- 无法强制解锁，需等待进程释放

#### 1.1.5 生效条件
- 修改后无需重启通达信
- 需在通达信中执行"刷新板块"操作 (快捷键 Ctrl+F5)
- 部分版本可能需要重新登录

|--------|------|------|
| 路径自动探测 | ✅ 成功 | 覆盖90%常见安装场景 |
| 文件读取解析 | ✅ 成功 | 正确提取板块名称和股票列表 |

### 2.3 数据流转设计

#### 2.3.1 读取流程
```
用户请求板块列表
    ↓
Flask路由接收请求
    ↓
BlockManager.list_custom_blocks()
    ↓
PathDetector.detect_tdx_path() → 获取安装路径
    ↓
BlockIndexParser.parse("block_gn.dat") → 解析板块索引
    ↓
返回 JSON: [{code, name, stock_count}, ...]
    ↓
前端渲染板块列表
```

#### 2.3.2 修改流程
```
用户提交修改请求
    ↓
Flask路由接收请求 (POST /api/blocks/{code}/stocks)
    ↓
ProcessMonitor.check_file_lock() → 检测文件占用
    ├─ 被占用 → 返回错误 (请关闭通达信)
    └─ 未占用 → 继续
    ↓
Backup.create_backup() → 创建备份
    ↓
BlockStockParser.serialize(stocks) → 序列化数据
    ↓
文件写入 (原子操作：先写临时文件再替换)
    ↓
验证写入结果 (重新读取比对)
    ├─ 失败 → 从备份恢复
    └─ 成功 → 返回成功响应
    ↓
前端提示保存成功 + 刷新建议
```

## 3. 接口设计

### 3.1 RESTful API

| 方法 | 路径 | 描述 | 请求参数 | 响应格式 |
|------|------|------|----------|----------|
| GET | `/api/tdx/path` | 获取通达信路径 | - | `{"path": "...", "found": true}` |
| GET | `/api/blocks` | 获取板块列表 | - | `[{code, name, count}, ...]` |
| GET | `/api/blocks/<code>` | 获取板块详情 | - | `{code, name, stocks: [...]}` |
| PUT | `/api/blocks/<code>` | 更新板块名称 | `{name: "..."}` | `{success: true}` |
| GET | `/api/blocks/<code>/stocks` | 获取成分股 | - | `[{code, name, market}, ...]` |
| PUT | `/api/blocks/<code>/stocks` | 更新成分股 | `{stocks: ["000001.SZ", ...]}` | `{success: true}` |
| POST | `/api/blocks` | 新建板块 | `{name: "..."}` | `{code: "BK0099", success: true}` |
| DELETE | `/api/blocks/<code>` | 删除板块 | - | `{success: true}` |
| GET | `/api/backups` | 获取备份列表 | - | `[{filename, timestamp, size}, ...]` |
| POST | `/api/backups/<filename>/restore` | 恢复备份 | - | `{success: true}` |

### 3.2 错误响应格式
```json
{
  "success": false,
  "error": {
    "code": "FILE_LOCKED",
    "message": "通达信正在运行，请关闭后重试",
    "details": "文件 C:\\new_hxtd\\T0002\\hq_cache\\block.dat 被占用"
  }
}
```

**错误码定义**：
- `PATH_NOT_FOUND`: 未找到通达信安装路径
- `FILE_LOCKED`: 文件被进程占用
- `PARSE_ERROR`: 文件格式解析失败
- `WRITE_FAILED`: 数据写入失败
- `BACKUP_FAILED`: 备份创建失败
- `INVALID_STOCK_CODE`: 股票代码格式错误

## 4. 前端设计

### 4.1 页面结构

#### 4.1.1 首页 (`/`)
**功能**：展示所有自定义板块
**UI组件**：
- 搜索框：按板块名称筛选
- 板块卡片列表：
  - 板块名称 (大字体)
  - 股票数量 (标签显示)
  - 操作按钮：[编辑] [删除]
- 新建板块按钮 (右上角)
- 状态栏：显示通达信路径、连接状态

#### 4.1.2 编辑页 (`/blocks/<code>/edit`)
**功能**：编辑板块成分股
**UI组件**：
- 板块信息区：
  - 板块名称 (可编辑)
  - 板块代码 (只读)
- 股票列表区：
  - 表格展示：代码 | 名称 | 市场 | 操作
  - 支持多选删除
  - 支持拖拽排序 (可选)
- 添加股票区：
  - 批量输入框 (支持粘贴多行代码)
  - 单只添加输入框 (带自动补全)
  - 从其他板块导入按钮
- 操作区：
  - [保存] 按钮 (带加载状态)
  - [取消] 按钮
  - 修改预览 (显示增减数量)

### 4.2 交互设计

#### 4.2.1 关键交互流程
1. **板块选择**：
   - 点击板块卡片 → 跳转编辑页
   - 双击板块名称 → 直接重命名

2. **股票添加**：
   - 输入代码 → 实时校验格式
   - 粘贴多行 → 自动分割并批量添加
   - 重复代码 → 自动去重提示

3. **保存确认**：
   - 点击保存 → 弹出确认对话框
   - 显示变更摘要 (+5只，-2只)
   - 确认后执行写入操作
   - 显示进度条和结果提示

4. **异常处理**：
   - 文件占用 → 红色警告 + "如何关闭通达信"帮助链接
   - 写入失败 → 显示错误详情 + "恢复备份"按钮
   - 网络错误 → 自动重试机制

### 4.3 技术选型

- **交互**：Vanilla JavaScript (无需构建工具)
- **HTTP客户端**：Fetch API
- **图标**：Bootstrap Icons

## 6. 风险应对策略

### 6.1 已识别风险及应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 文件格式版本差异 | 中 | 高 | 建立格式规则配置化；提供手动指定版本功能 |
| 通达信加密数据 | 低 | 高 | 提前调研主流版本；准备逆向分析工具 |
| 并发写入冲突 | 低 | 中 | 文件锁检测 + 乐观锁机制 |
| GBK编码特殊字符 | 中 | 中 | 建立字符映射表；异常字符替换策略 |
| 大板块性能问题 | 低 | 低 | 分页加载；增量更新优化 |

### 6.2 降级方案
- **路径探测失败**：提供手动输入路径界面
- **自动解析失败**：提供十六进制编辑器模式 (高级用户)
- **Web服务不可用**：准备命令行备用工具
