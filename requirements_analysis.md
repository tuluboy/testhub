# 通达信自定义板块管理工具 - 需求分析文档

## 1. 项目概述

### 1.1 项目背景
通达信是国内广泛使用的股票行情分析软件，用户常需要管理自定义板块（如自选股、行业分类、主题概念等）。目前修改自定义板块内容需要通过通达信软件界面操作，批量修改或程序化管理较为不便。本项目旨在开发一个 Python 小工具，直接操作通达信底层数据文件，提供便捷的自定义板块管理功能。

### 1.2 项目目标
- 自动定位本机通达信软件安装目录
- 解析通达信自定义板块数据文件格式
- 提供 Web 界面展示和编辑自定义板块
- 实现板块内容的增删改查操作
- 确保数据修改后能在通达信软件中正确显示

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 通达信安装路径自动探测
- **功能描述**：自动扫描本机可能的通达信安装位置
- **探测范围**：
  - 默认安装路径（如 `C:\new_hxtd`、`C:\Program Files\通达信`）
  - 注册表查询（Windows 系统）
  - 常见安装目录扫描
- **输出结果**：返回通达信主目录路径，若未找到则提示用户手动指定

#### 2.1.2 自定义板块数据文件识别
- **目标文件**：定位存储自定义板块数据的文件
  - 主要文件：`T0002\hq_cache\block.dat`（板块索引）
  - 关联文件：`T0002\hq_cache\block_cfg.dat`（板块配置）
  - 用户自定义板块文件：`T0002\block_gn.dat`、`T0002\block_zs.dat` 等
- **文件格式解析**：
  - 解析二进制/文本混合格式
  - 提取板块名称、板块代码、包含股票代码列表

#### 2.1.3 Web 管理界面
- **首页（板块列表页）**：
  - 展示所有自定义板块名称列表
  - 显示每个板块包含的股票数量
  - 提供"编辑"按钮入口
  - 支持板块搜索/筛选功能
  
- **编辑页（板块内容编辑）**：
  - 显示选中板块的详细信息（名称、代码）
  - 展示当前板块包含的所有股票代码及名称
  - 提供股票添加功能（支持批量粘贴股票代码）
  - 提供股票删除功能（单选/多选删除）
  - 提供股票排序功能（可选）
  - 实时显示修改预览
  - "保存"按钮提交修改
  - "取消"按钮放弃修改

#### 2.1.4 数据持久化
- **保存机制**：
  - 修改后直接写入通达信数据文件
  - 操作前自动备份原文件（防止数据损坏）
  - 验证写入数据的格式正确性
- **一致性保障**：
  - 确保通达信软件未运行时进行文件操作
  - 提供"刷新"功能重新读取最新数据

### 2.2 辅助功能

#### 2.2.1 数据备份与恢复
- 每次修改前自动创建备份文件（带时间戳）
- 提供备份文件列表和恢复功能

#### 2.2.2 操作日志
- 记录所有修改操作（时间、板块、操作类型）
- 日志可查看和导出

#### 2.2.3 错误处理
- 文件占用检测（通达信运行时提示关闭）
- 数据格式异常处理
- 操作失败回滚机制

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2 秒（板块数量 < 100 时）
- 保存操作响应时间 < 3 秒
- 支持同时管理最多 200 个自定义板块

### 3.2 兼容性要求
- **操作系统**：Windows 7/10/11（通达信主要运行平台）
- **通达信版本**：兼容主流版本（需调研具体文件格式差异）
- **Python 版本**：Python 3.8+

### 3.3 安全要求
- 仅操作本地文件，不涉及网络传输
- 敏感操作（如删除整个板块）需二次确认
- 不修改通达信程序文件，仅操作数据文件

### 3.4 可用性要求
- 界面简洁直观，无需专业培训即可使用
- 提供操作提示和帮助文档
- 支持键盘快捷键操作

## 4. 技术架构

### 4.1 技术栈选择
- **后端**：Python 3.8+
  - Web 框架：Flask（轻量级，易于部署）
  - 文件操作：原生 Python + 结构化数据处理库
  - 路径探测：`os`、`winreg`（Windows 注册表）
- **前端**：
  - HTML5 + CSS3 + JavaScript（原生或轻量框架）
  - UI 组件：Bootstrap 5（快速构建响应式界面）
  - 交互：jQuery 或 Vanilla JS
- **数据存储**：
  - 临时存储：内存变量
  - 持久化：通达信原生数据文件
  - 备份文件：本地文件系统

### 4.2 系统架构
```
用户界面 (浏览器) 
    ↓
Flask Web 服务器 (Python)
    ↓
业务逻辑层
    ├── 路径探测模块
    ├── 文件解析模块
    ├── 数据修改模块
    └── 备份恢复模块
    ↓
文件系统 (通达信数据文件)
```

## 5. 数据格式调研（待确认）

### 5.1 关键问题
在正式开发前，需要明确以下技术细节：
1. **板块数据文件确切路径**：不同版本通达信可能有所不同
2. **文件格式规范**：
   - 是纯文本、二进制还是混合格式？
   - 股票代码的编码方式（6 位数字 + 市场标识？）
   - 板块名称的存储编码（GBK/UTF-8？）
3. **文件锁定机制**：通达信运行时是否独占文件？
4. **生效条件**：修改后是否需要重启通达信或刷新缓存？

### 5.2 调研计划
- 收集不同版本通达信的样本数据文件
- 使用十六进制编辑器分析文件结构
- 测试修改后在通达信中的实际效果
- 建立文件格式解析规则库

## 7. 风险与应对

| 风险点 | 影响程度 | 应对措施 |
|--------|----------|----------|
| 通达信文件格式加密或复杂 | 高 | 提前调研，必要时逆向分析；提供手动配置入口 |
| 文件被通达信进程占用 | 中 | 检测进程状态，提示用户关闭；尝试强制解锁 |
| 修改后通达信不识别 | 高 | 严格遵循原格式；提供恢复备份功能 |
| 不同版本格式差异 | 中 | 建立版本适配机制；支持手动指定格式规则 |
